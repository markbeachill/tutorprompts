Markdown guide

What Markdown is, why the toolkit uses it, and how to view or edit Markdown files safely.

1. What Markdown is

Markdown is a plain-text format that uses simple symbols for structure. A Markdown file can include headings, lists, links, emphasis and code blocks while still being readable as ordinary text.

The file ending .md means Markdown. You do not need specialist software to open it.

2. Why the toolkit uses Markdown for AI

Markdown is useful for AI prompts because it makes structure visible. Headings, lists and code blocks help separate instructions, menus, examples and warnings.

  • It is lightweight and portable.
  • It avoids hidden formatting from Word or PDF files.
  • It is easy to copy, upload, version and inspect.
  • It helps AI tools distinguish sections of a prompt library.

3. Viewing and editing Markdown with GitHub and GitHub Pages

GitHub can preview Markdown files in the browser. If you have permission to edit a repository, GitHub can also be used to edit Markdown files directly.

GitHub Pages is different: it publishes the website. It is not normally where you edit the Markdown files themselves.

You can also download a raw .md file and edit it locally on your own computer.

4. Using LibreOffice

LibreOffice Writer can open plain-text Markdown files, although it may show the Markdown symbols rather than a formatted preview.

If you edit a Markdown file in LibreOffice, keep the plain-text symbols and take care when saving. Save as plain text or Markdown if you want to keep a .md file, rather than accidentally converting it into .odt or .docx.

LibreOffice can be useful when you want a local/offline tool rather than an online converter.

5. Other Markdown tools and online converters

There are many Markdown editors and previewers. Desktop tools can be useful if you want to see a formatted preview while editing.

Be careful with online Markdown converters. They may send your text to an external service. Avoid using them for confidential, personal, student, assessment, research, client or workplace material unless you understand and accept the privacy implications.

For sensitive material, prefer local or offline tools.
