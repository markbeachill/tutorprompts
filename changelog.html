Changelog

Version history.

A short record of major changes to the toolkit files and site.

Site v2.11 / Libraries v2.4 / Testing pack v2.8

  • Updated WT3 so simple mistakes can be corrected directly while complex clause-level issues are explained rather than turned into near-complete replacement sentences.
  • Added a plain-English grammar-note test to reduce unnecessary terminology in WT3 explanations.
  • Added a WT3 challenge-response protocol so valid student pushback is acknowledged explicitly before the mistake list is revised.
  • Updated the audit prompt, educator guide and step-by-step test cards with WT3 regression checks.

Site v2.10 / Libraries v2.3 / Testing pack v2.7

  • Updated WT5 so style-and-clarity feedback uses location, issue, reader effect, move to make, and student-owned revision prompts.
  • Reduced authorship drift by removing polished replacement sentences as the default WT5 output pattern.
  • Added WT5 vague-meaning guidance so the tool asks clarification questions rather than specifying the student's research focus on their behalf.
  • Added a strong-passage teaching pattern so students learn from parts of their own writing that already work well.
  • Updated the audit prompt and step-by-step test cards with WT5 regression checks.

Site v2.9 / Libraries v2.2 / Testing pack v2.6

  • Updated ST1 so it checks central claim clarity before diagnosing paragraph development, evidence, links or polish.
  • Added guidance to distinguish an unclear or unformed central claim from merely thin development.
  • Tightened ST1 follow-up boundaries so examples use fictional topics rather than near-usable sentences based on the student's own source material.
  • Added guidance on handling marker, tutor or supervisor feedback as evidence of reader confusion, not as a marker-response service.
  • Updated the audit prompt and ST1 test card so this central-claim issue is checked in future regression testing.

Site v2.8 / Libraries v2.1 / Testing pack v2.5

  • Fixed version consistency across library front matter, visible headers and download folders.
  • Removed duplicate global AI-behaviour guidance from the prompt libraries.
  • Consolidated repeated precision-before-polish wording in clinic-style guidance.
  • Fixed audit-pack instructions so step-by-step cards point to the correct audit prompt version.
  • Updated the homepage from the current student-facing version and removed developer folder details from student-facing copy.

Site v2.6 / Libraries v2.0 / Testing pack v2.4

  • Added version-folder structure: current downloads live in prompt-libraries/latest/ and audit-library/latest/.
  • Added fixed release copies in prompt-libraries/v2.0/ and audit-library/v2.4/.

Libraries v2.0 — WT2 paragraph logic update

  • Updated WT2 Single Paragraph Analysis so it starts with the paragraph's chain of ideas, missing links and “so what?” questions.
  • Added controlled modelling rules for paragraph logic examples.
  • Updated WT2 audit tests and criteria.

Libraries v1.9 — Precision before polish

  • Added meaning-preservation rules so the tools do not silently swap key terms for more academic-sounding alternatives.
  • Updated WT1 and WT5 to guard against academicising that changes meaning.

Earlier releases

Earlier versions created the five mini libraries, the master library, the testing pack, tool codes, privacy/responsibility notes, language settings and the multi-page website.
