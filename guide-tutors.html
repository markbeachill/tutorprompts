Tutor and teacher guide

How should educators recommend the toolkit safely?

Guidance for tutors, teachers, supervisors and learning developers who want to use the toolkit for formative support without turning it into a ghost-writing or automated-marking system.

Recommended starting point

Use first for low-stakes formative work.

The safest initial use is draft-stage support: helping students understand problems, practise sub-skills, revise their own work and interpret feedback. Do not position the toolkit as an automated marker or a substitute for teacher judgement.

Suitable uses

  • Formative feedback on draft extracts.
  • Paragraph and structure diagnosis before revision.
  • Turning tutor comments into student action plans.
  • Practising repeated grammar, clarity or argument problems.
  • Supporting transparent AI-use records.
  • Helping students prepare for supervision or viva-style questioning.

Unsuitable uses

  • Automated marking or grading.
  • Generating first drafts for assessment.
  • Correcting confidential student work in public AI tools without checking policy.
  • Using AI output as evidence that a student has met a learning outcome.
  • Bypassing institutional assessment, privacy, safeguarding or data-protection rules.

Before recommending the toolkit

  1. Check your course, department or institution rules on AI use.
  2. Decide which tasks are acceptable for toolkit use and which are not.
  3. Tell students what they should not paste or upload.
  4. Choose whether students should use mini libraries or the master library.
  5. Run relevant tests from the testing pack before recommending a new version at scale.
  6. Explain when students should use SW3 to record AI use.

How to introduce it to students

A useful framing is:

This toolkit can help you understand and revise your own work. It should not write the work for you. Bring it something you have tried, use the feedback to make your own decisions, and follow the AI-use rules for this course.

Using the testing pack

The testing pack is for educators, tutors, learning developers and maintainers who want to check behaviour before recommending the toolkit. Use smoke tests for basic operation, regression tests for known risks, and adversarial tests for boundary pressure.
