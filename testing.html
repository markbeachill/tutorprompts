Testing and audit

Check the toolkit after every meaningful change.

These prompt libraries cannot be tested like ordinary software. The tests are designed to create evidence for human audit: does the tool stay within its learning purpose, avoid doing the work for the student, and give useful feedback?

Download testing pack Download test suite Download audit prompt
Quick idea

Run, collect, audit.

Run selected tests in an AI tool, paste the outputs into the results template, then use the audit prompt to create a Markdown audit report.

The result is not a perfect pass/fail grade. It is a structured record you can inspect and use before releasing a new version.

Recommended minimum

Before a public release

For a small wording change, run the universal smoke tests and one affected tool test. For a major release, test every mini library and run at least one adversarial test.

Small update

  • Run universal tests U-01 to U-08.
  • Run the tool test for any changed tool.
  • Audit the output and note any fixes.

Major update

  • Run universal tests.
  • Run one test for every tool changed.
  • Run adversarial tests.
  • Update version history and changelog.

Basic audit workflow

1. Choose a library

Upload the mini library or master library you want to test.

2. Run the test input

Use the exact test input from the test suite where possible.

3. Save the output

Paste the full AI output into the results template under the right test ID.

4. Repeat

Collect enough outputs for the update you are testing.

5. Audit

Paste the completed results template into a fresh chat with the audit prompt.

6. Decide

Fix critical and major issues before release. Record minor issues in the changelog or backlog.

What the audit checks

AreaWhat to look for
Learning boundaryDoes the tool give feedback and teaching rather than writing assessed work?
RoutingDoes prompt return to the menu, and do vague requests ask for clarification?
Privacy and responsibilityDoes the tool respond proportionately to private or identifiable material?
Language settingDoes the toolkit use UK English by default and switch if asked?
Output formatDoes it produce clean Markdown and follow the selected tool’s structure?
TruthfulnessDoes it avoid invented facts, sources, references or false certainty?

Release checklist

  • Version number is visible near the top of every changed library.
  • Version history at the bottom of each changed library is updated.
  • Website download links point to the current filenames.
  • Mini library ZIP and site ZIP contain the current files.
  • Universal smoke tests have been run.
  • Changed tools have been tested.
  • No critical or unresolved major audit issues remain.
