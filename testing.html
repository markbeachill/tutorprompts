Testing and audit.

Use this area to check whether the prompt libraries behave like learning tools, not answer machines. The tests are practical, step by step, and designed for educators rather than software testers.

Download testing pack Download test cards Download audit prompt

Start here

The testing pack is for educators, tutors, learning developers and toolkit maintainers who want to check the behaviour of the prompts after edits.

You do not need to be a software tester. Choose one test card, run the prompt, save the output, then use the audit prompt to review it.

Simple testing workflow

  1. Choose one test card.
  2. Open a new AI chat.
  3. Upload the prompt library named in the test card.
  4. Type prompt and choose the named tool.
  5. Paste the copyable test input.
  6. Save the AI output using the suggested filename.
  7. Open a second AI chat.
  8. Upload or paste the audit prompt.
  9. Choose the matching audit from the audit menu.
  10. Paste the saved output and save the audit response.

Plain-English testing terms

Smoke test

A quick check that the basics work: the menu appears, routing works, and the tool responds to simple input.

Regression test

A check that something you fixed has not broken again in a later version.

Adversarial test

A check using a vague, risky or boundary-pushing request, such as asking the tool to rewrite work for submission.

Tool types matter

Interactive tutoring tools

These should keep the student active. They ask the student to think, choose, answer, revise or attempt something. They should not give polished final wording too early.

Examples: WT1, WT2, WT4, AT10, RP4, RP5.

Full review tools

These can produce a full structured review. They should give feedback and priorities, but should not rewrite whole sections or produce submission-ready replacement prose.

Examples: WT3, WT5, ST1, ST2, ST3, RP3, SW1.

Suggested minimum before release

  • Run the universal menu and language-setting tests.
  • Run at least one adversarial test.
  • Run WT1, including the follow-up turn about sounding academic.
  • Run WT2, including the follow-up turn about connecting paragraph ideas.
  • Run one changed tool from each affected mini library.
  • Record the result in the test log.
