# Site update v2.23 — site and testing/audit maintenance

Site package: v2.23  
Prompt-library suite: v2.4  
Testing/audit pack: v2.9

## Summary

This release updates the testing/audit pack and makes site-copy refinements from recent review feedback.

## Testing/audit changes

- Added clearer two-chat workflow guidance so testers understand why the tool test and audit should happen in separate AI chats.
- Removed ambiguous hard-coded version numbers from suggested output filenames; versions are now recorded in the test log.
- Added AI tool/model and plan/access level fields to the test log.
- Added clearer PASS / MINOR ISSUE / MAJOR ISSUE / CRITICAL ISSUE / NOT TESTABLE boundaries to the audit prompt.
- Added guidance for marking non-applicable evidence-table criteria as N/A.
- Added what-to-do-if-a-test-fails guidance.
- Strengthened SW3 AI-Use Record test input and audit focus.
- Cross-referenced U1–U4 and A1–A2 universal/adversarial tests.

## Site-copy changes

- Improved the Examples page with more varied placeholder examples and a short dialogue example.
- Added onward navigation from Examples to Student Help, Tools and downloads.
- Added tool-library code guidance to the Student guide.
- Expanded the first-draft-generator guide with a section on why the first struggle matters.
- Added Typora to the Markdown guide.
- Added small refinements to Teaching approach, Why educators, Tutor/teacher guide, Customising prompts, Other AI resources and Setup guide pages.
- Added dependency/overlap notes to the Tools page.

No prompt-library behaviour changed.
